## Genuine feature gaps & partials (the actionable output)

Everything else is either ✅ built or ⚙️ live-gated (credentials/provisioning, covered in the prior "outstanding" summary). These are the items where **code is missing or incomplete vs V2.4**. **Correction:** item 2 (Offline/PWA) was originally listed as a gap in error — WO-101 built it; only server-side hardening remains. So the genuine *feature* gaps are **4** (Template Library, Knowledge AI-capture, Calendar sync, Accounting breadth) plus **1 hardening** WO (offline sync). All five are scoped in `PLAN-V2.4-GAP-CLOSURE.md` as WO-121…WO-125.

1. **❌ Template Library** (§11; §23 layer 17) — "AI suggests new templates from completed engagements; admin approves via learning update process." No model, migration, service, or UI. *Not built.*
2. **✅ Offline mode / PWA** (§7; §27) — **built by WO-101** (corrected from an earlier audit error): `public/manifest.webmanifest`, `public/sw.js` (background sync `portal-offline-sync`), `public/offline.html`, encrypted IndexedDB queue in `resources/js/lib/portal-offline.ts` (`crypto.subtle`, dedupe keys, `queueQuestionnaireSubmission`/`queueDocumentUpload`/`flushPortalOfflineQueue`), proven by `tests/Feature/Portal/OfflinePwaTest.php`. The client/browser offline surface is complete. *Remaining work is server-side hardening only* — DB-enforced sync idempotency, scanned upload replay (`SecureFileWriter`), re-auth/RLS on sync — tracked as **WO-122** in `PLAN-V2.4-GAP-CLOSURE.md`, not a fresh build.
3. **⚠️ Advisor Knowledge Base — AI-assisted capture** (§11; §23 layer 16) — manual entry is built (WO-27); the "AI-assisted capture after each engagement" half is not.
4. **⚠️ Calendar two-way sync** (§21; §25) — `GoogleCalendarClient`/`MicrosoftGraphClient` exist as **Contract + Fake only** (no live client, no sync logic/UI). Scaffold, not working sync.
5. **⚠️ Accounting breadth** (§25) — Xero/MYOB/QuickBooks are built (WO-37); **Sage, Figured, Workflowmax** are named-but-empty scaffolds.

## Stack deviations (accepted, documented)
- **Backend: Laravel (PHP)** — spec recommended Node/Express or Python/FastAPI; Laravel is the locked choice (CLAUDE.md stack table). `AiClient`/`ResilientHttp`/`AuditWriter`/`KeyEnvelope` satisfy the spec's architectural intent.
- **Auth: Laravel Fortify + custom MFA/invite** — spec recommended Auth0/Cognito; the custom layer meets the §4 security requirements (invite-only, mandatory MFA, step-up).

These honour the spec's *requirements* (the stack section is explicitly "recommendations… validated against developer expertise").

## Gap-closure outcomes

Everything else is either ✅ built or ⚙️ live-gated (credentials/provisioning, covered in the prior "outstanding" summary). **Correction:** item 2 (Offline/PWA) was originally listed as a gap in error — WO-101 built it; WO-122 closes the server-side hardening. WO-121...WO-125 are complete; no V2.4 feature gaps remain in this gap-closure track.

1. **✅ Template Library** (§11; §23) — **built by WO-121**: `templates` library, `TemplateSuggestionLayer` registry ID 33, governed learning candidates, `TemplateImplementer`, rollback to dormant draft, permissions, internal advisor UI.
2. **✅ Offline mode / PWA** (§7; §27) — **built by WO-101 and hardened by WO-122**: installable portal shell, encrypted IndexedDB queue, background sync, queued-client replay headers, DB-enforced idempotency, request fingerprints, scanned upload replay, sync auth JSON normalization, explicit logout clearing.
3. **✅ Advisor Knowledge Base — AI-assisted capture** (§11; §23 layer 16) — **built by WO-123**: offboarding completion and a manual advisor action create advisor-owned drafts in `knowledge_entry_drafts`; advisors review/edit/accept/discard, and drafts never enter the live `knowledge_entries` index until accepted.
4. **✅ Calendar two-way sync** (§21; §25) — **built by WO-124**: `calendar_connections` and `calendar_event_mappings`, Google/Microsoft live+fallback clients, encrypted access/refresh tokens, OAuth connect/revoke, fixture push/pull, idempotent incremental sync, settings UI, synced meeting surfacing, RLS, resilience ledger fallback, and crypto rewrap coverage.
5. **✅ Accounting breadth** (§25) — **built by WO-125**: Sage, Figured, and WorkflowMax now mirror Xero/MYOB/QuickBooks with provider contracts, fake/live/fallback clients, `IntegrationServiceProvider` bindings, `AccountingClientResolver` match arms, OAuth scopes, fixture snapshots, settings-driven feature flags, and client-detail connect options.

## Stack deviations (accepted, documented)
- **Backend: Laravel (PHP)** — spec recommended Node/Express or Python/FastAPI; Laravel is the locked choice (CLAUDE.md stack table). `AiClient`/`ResilientHttp`/`AuditWriter`/`KeyEnvelope` satisfy the spec's architectural intent.
- **Auth: Laravel Fortify + custom MFA/invite** — spec recommended Auth0/Cognito; the custom layer meets the §4 security requirements (invite-only, mandatory MFA, step-up).

These honour the spec's *requirements* (the stack section is explicitly "recommendations… validated against developer expertise").
